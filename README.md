# Intelligent Exploitation of Data from a Leak Test Machine Using Machine Learning for Quality Decision Support in an Industry 4.0 Context

## 📌 Overview

This project explores how **machine learning** can be used to intelligently exploit data generated by a **leak test machine**, a device widely used in manufacturing to detect sealing defects and leaks in products such as containers, assemblies, or components.

Instead of relying only on the machine's basic **pass/fail** output, this project leverages the underlying test data (pressure curves, decay rates, timing measurements, sensor logs, etc.) to build predictive and analytical models that support **smarter, faster, and more reliable quality decisions**.

The work is framed within an **Industry 4.0** context, where interconnected machines, sensors, and intelligent software systems collaborate to enable real-time, data-driven decision-making on the production line.

---

## 🎯 Objectives

- Collect, clean, and structure raw data produced by the leak test machine.
- Explore and visualize patterns in leak test measurements.
- Apply machine learning models to:
  - Predict product failures/defects.
  - Classify types of leaks or sealing issues.
  - Detect anomalies or early warning signs of machine/process drift.
- Provide quality engineers with clear, actionable decision support (beyond simple pass/fail).
- Demonstrate how legacy test equipment can be integrated into an Industry 4.0 / smart manufacturing workflow.

---

## 🏭 Context: Why This Matters

Leak test machines already generate large volumes of data, but this data is often underused — reduced to a binary outcome and discarded. This project aims to unlock the hidden value in that data, transforming the leak test machine from a simple checkpoint into an **intelligent data source** that contributes to continuous quality improvement.

---

## 🧠 Approach / Methodology

1. **Data Acquisition** – Extracting raw signal/measurement data from the leak test machine.
2. **Data Preprocessing** – Cleaning, normalizing, and structuring the data for analysis.
3. **Exploratory Data Analysis (EDA)** – Understanding trends, distributions, and correlations in the data.
4. **Model Development** – Training machine learning models (classificationn) suited to the problem.
5. **Model Evaluation** – Assessing performance using relevant metrics (accuracy, precision, recall, F1-score, etc.).
6. **Decision Support Integration** – Presenting model outputs in a way that supports quality decisions in real time.
